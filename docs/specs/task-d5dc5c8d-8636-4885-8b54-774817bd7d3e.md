# Fix Upload Timeout and Progress During PhotoPrism Processing

Uploading many photos fails or hangs indefinitely during the "Zpracování v PhotoPrismu..." phase because `ProcessUpload` uses `http.DefaultClient` (no timeout) and provides no progress feedback.

## Root Cause

1. `internal/photoprism/http.go` uses `http.DefaultClient` which has no timeout. `ProcessUpload` calls `PUT /users/{uid}/upload/{token}` which triggers PhotoPrism's import + indexing pipeline (thumbnail generation, metadata extraction, face detection). For large uploads this can take many minutes and the HTTP call blocks indefinitely.
2. `processUploadTokens` in `internal/web/handlers/upload_job.go` runs 8 parallel process calls (`UploadProcessConcurrency = 8`). On a Raspberry Pi this is too aggressive — PhotoPrism struggles with concurrent indexing.
3. No SSE progress events are sent during the processing phase — the frontend shows a static "Processing..." message with no indication of progress.

## Requirements

### 1. Add HTTP client timeout

Create a dedicated `http.Client` in the PhotoPrism struct with a configurable timeout (default 5 minutes per request). Use this client for all API calls instead of `http.DefaultClient`.

For `ProcessUpload` specifically, use a longer timeout (10 minutes) since indexing is slow. Consider making this configurable via env var `PHOTOPRISM_PROCESS_TIMEOUT`.

### 2. Reduce processing concurrency

Change `UploadProcessConcurrency` from 8 to 2 (or make it configurable). PhotoPrism on a Pi cannot handle 8 concurrent import jobs.

### 3. Add processing progress events

In `processUploadTokens`, send SSE progress events as each token is processed:
```
{ type: "processing_upload", data: { current: 3, total: 10 } }
```

This requires passing the `*UploadJob` into `processUploadTokens` so it can call `job.SendEvent`.

### 4. Frontend progress display

Update the upload progress UI to show processing progress: "Zpracování v PhotoPrismu 3/10..." instead of a static message.

### 5. Graceful timeout handling

If a `ProcessUpload` call times out:
- Log a warning but don't fail the entire upload job
- Continue with the next token
- At the end, report how many tokens were successfully processed vs timed out

## Files to modify

- `internal/photoprism/photoprism.go` — add `http.Client` with timeout to struct
- `internal/photoprism/http.go` — use the struct's client instead of `http.DefaultClient`
- `internal/photoprism/upload.go` — use longer timeout for ProcessUpload
- `internal/web/handlers/upload_job.go` — pass job to processUploadTokens, send progress events
- `internal/constants/handlers.go` — reduce UploadProcessConcurrency to 2
- `web/src/pages/Upload/hooks/useUploadJob.ts` — handle processing progress data
- `web/src/pages/Upload/index.tsx` — display processing progress count
