## Problem

In the Book Editor, photo thumbnails show `FileName` (e.g. `2025/05/20250510_065959_F69BA7C1.jpg` — the renamed path in PhotoPrism storage). The user wants to see `OriginalName` instead (e.g. `20250510_085959.jpg` — the original filename at upload time), which is more meaningful.

## Solution

Change the backend `fetchPhotoNames` function to return `OriginalName` instead of (or in addition to) `FileName`.

### Backend change

In `internal/web/handlers/books.go`, function `fetchPhotoNames` (~line 1686):

Current code:
```go
names[p.UID] = photoNameInfo{Title: p.Title, FileName: p.FileName}
```

Change to prefer `OriginalName`, falling back to `FileName`:
```go
fn := p.OriginalName
if fn == "" {
    fn = p.FileName
}
names[p.UID] = photoNameInfo{Title: p.Title, FileName: fn}
```

This ensures photos that don't have an `OriginalName` (e.g. indexed directly, not uploaded) still show something useful.

### No frontend changes needed

The frontend already displays whatever `file_name` the API returns — no changes needed in `PhotoInfoOverlay.tsx`, `PageSlot.tsx`, `SectionPhotoPool.tsx`, etc.

### Verify

The `OriginalName` field already exists on the `Photo` struct in `internal/photoprism/types.go` (line 49). Confirmed it's populated by PhotoPrism API.
