# Add AI Text Check and Rewrite to TextSlotDialog

Add the same AI text validation tools that exist in `PhotoDescriptionDialog` to the `TextSlotDialog` component used for text-only page slots in the book editor.

## Context

The `PhotoDescriptionDialog` (`web/src/pages/BookEditor/PhotoDescriptionDialog.tsx`) already has two AI buttons:
1. **"Kontrola textu"** — spell check, diacritics, grammar + readability score (calls `POST /api/v1/text/check`)
2. **"Upravit délku"** — rewrite text shorter/longer with a select dropdown (calls `POST /api/v1/text/rewrite`)

Both show results in collapsible panels with an "Přijmout" (Accept) button to replace the textarea content.

The `TextSlotDialog` is defined inline within `PagesTab.tsx` and provides a split-pane markdown editor (textarea + live preview) with a formatting toolbar (H1, H2, Bold, Italic, Lists). This must be preserved.

## Requirements

1. Add the same two AI buttons ("Kontrola textu" and "Upravit délku" with length selector) to the `TextSlotDialog`
2. Place them in a row below the formatting toolbar or below the textarea, visually separated
3. AI result panels (corrections list, readability score, rewritten text) appear below the editor area
4. "Přijmout" button replaces the textarea content with the AI suggestion
5. Preserve the existing markdown toolbar and live preview — do not remove or break them
6. Both buttons disabled when textarea is empty or while AI is loading
7. Use the existing `checkText` and `rewriteText` API client functions from `web/src/api/client.ts`
8. Use existing i18n keys from `books.editor.*` (textCheck, adjustLength, checking, rewriting, accept, dismiss, textOk, readability, correctedText, changes, rewrittenText, muchShorter, shorter, longer, muchLonger)

## Implementation Notes

- The TextSlotDialog logic is currently inline in `PagesTab.tsx`. It may be cleaner to extract it to its own file (`TextSlotDialog.tsx`), but this is optional — inline is fine if it stays readable.
- Reuse the same pattern from `PhotoDescriptionDialog.tsx` for the AI state management (checking, checkResult, checkError, rewriting, rewriteResult, rewriteError, targetLength).
- The AI operates on the raw markdown text in the textarea, not the rendered HTML preview.
