# Cache AI Text Check Results

AI text operations cost money. If a user checks the same text twice, the result should be served from cache instead of calling the API again.

## Requirements

### 1. Backend cache

Add an in-memory cache for text check and rewrite results in the text handler (`internal/web/handlers/text.go`).

Cache key: hash of the input text (+ target_length for rewrite). Use a simple SHA-256 or similar hash of the request parameters.

Cache behavior:
- Before calling OpenAI, check if the same input was already processed
- If cached, return the cached result immediately (with `cost_czk: 0` since no API call was made)
- If not cached, call OpenAI, store the result, and return it
- Cache lives in memory (no database persistence needed) — it resets on server restart, which is fine
- No TTL needed — the cache is keyed on exact text content, so if the text changes, it's a cache miss

### 2. Frontend indication

When a cached result is returned (identifiable by `cost_czk: 0` or an explicit `cached: true` field in the response), show a subtle indicator in the result panel, e.g. "(z cache)" / "(cached)" next to the cost line, so the user understands why it was free.

Add the `cached` boolean field to both API responses and use it in the frontend for display.

### 3. i18n

Add key:
- `books.editor.cachedResult`: "z cache" (cs) / "cached" (en)
