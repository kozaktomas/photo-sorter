## Problém

PDF export fotoknihy nefunguje na vývojovém serveru (na produkci funguje). LuaLaTeX hlásí chybu:

```
luaotfload | resolve : sequence of 3 lookups yielded nothing appropriate.
! Font \TU/SourceSans3(0)/m/n/14.4=SourceSans3-Regular:mode=harf;language=dflt;
+tlig; at 14.4pt not loadable: metric data not found or bad.
```

Font **Source Sans 3** není nalezen. Pravděpodobně chybí nainstalovaný font v dev prostředí (Docker kontejner nebo host).

## Diagnóza

1. Zjisti, kde běží lualatex na dev serveru — přímo na hostu, nebo v Docker kontejneru?
2. Zkontroluj, zda je font Source Sans 3 nainstalovaný:
   ```bash
   fc-list | grep -i "source sans"
   luaotfload-tool --find="SourceSans3-Regular"
   ```
3. Podívej se do `internal/latex/` jak se font konfiguruje v LaTeX šablonách.
4. Porovnej s produkčním prostředím (Dockerfile, docker-compose) — jak je font tam nainstalován.

## Pravděpodobná oprava

- Nainstalovat font Source Sans 3 do dev prostředí (buď systémový font, nebo TeX font balíček)
- Možnosti:
  - `apt install fonts-source-sans-pro` nebo stáhnout Source Sans 3 z Google Fonts
  - Nebo TeX balíček: `tlmgr install sourcesanspro`
  - Po instalaci: `luaotfload-tool --update` pro rebuild font cache

## Kontext

- Font Source Sans 3 byl přidán jako secondary heading font v commitu `0202e2f` ("Add Source Sans 3 as secondary heading font for book text slots")
- Na produkci funguje — problém je specifický pro dev prostředí
- Viz `internal/latex/` pro LaTeX šablony a font konfiguraci
