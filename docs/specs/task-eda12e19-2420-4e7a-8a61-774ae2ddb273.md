## Problem

In the PDF export, the H1 chapter-colored background box overflows vertically beyond the heading text. Instead of wrapping just the H1 line, the colored box extends downward through the body text and into the photo slot below it. Visible in `4_landscape` layout where text slot is in top-left and photo is directly below.

See attached PDF: "SDH VESELICE" heading has an olive/dark-yellow background box that bleeds vertically past the text area into the bottom-left photo.

## Root Cause

In `internal/latex/markdown.go`, `formatHeading()` (~line 209-220):

```go
inner := fmt.Sprintf(
    `\vspace{10pt}\hspace{16pt}%s%s\textcolor{%s}{%s}\hspace{16pt}\vspace{10pt}`,
    sizeCmd, fontCmd, textColor, text,
)
box := fmt.Sprintf(
    `{\fboxsep=0pt\noindent\hspace{-4mm}\colorbox{chaptercolor}{\parbox{\dimexpr\linewidth+8mm}{%s}}}`,
    inner,
)
```

The `\colorbox` wraps a `\parbox` which is placed inside the text slot's outer `\parbox` (in the TikZ node from `book.tex` line 94). The combination of nested parboxes and `\vspace` commands likely causes the inner `\parbox` to not properly constrain its height, allowing the colored background to overflow.

Possible specific causes:
1. The inner `\parbox` has no explicit height constraint — it uses natural height, but `\vspace{10pt}` commands inside may interact badly with the outer parbox
2. The `\colorbox` may not be properly clipped by the TikZ node's boundaries
3. The `\fboxsep=0pt` combined with `\noindent` and `\hspace{-4mm}` bleed may cause the box to escape its container

## Solution

Fix the LaTeX rendering so the colored box wraps only the H1 heading line with its padding, not the entire text slot.

### Approach 1: Use a fixed-height construct

Replace the `\parbox{\dimexpr\linewidth+8mm}` with a construct that constrains height to the heading content:

```latex
{\fboxsep=0pt\noindent\hspace{-<bleed>mm}\colorbox{chaptercolor}{%
  \makebox[\dimexpr\linewidth+<totalbleed>mm][l]{%
    \rule{0pt}{28pt}% strut for consistent height
    \hspace{16pt}<sizeCmd><fontCmd>\textcolor{<color>}{<text>}\hspace{16pt}%
  }%
}\par\vspace{4mm}}
```

Using `\makebox` instead of `\parbox` forces a single-line box that won't expand vertically.

### Approach 2: Add explicit height to parbox

```latex
\colorbox{chaptercolor}{\parbox[t][\totalheight]{\dimexpr\linewidth+8mm}{...}}
```

The `[t]` alignment anchor and `[\totalheight]` explicit height may contain the overflow.

### Approach 3: Wrap in a minipage with clipping

Use `\begin{minipage}` with explicit height instead of `\parbox`, which has better containment behavior.

### Recommendation

Try Approach 1 first (`\makebox` instead of `\parbox`) — it's the simplest and most robust since H1 headings are typically single-line. If multi-line H1s are needed, use Approach 2.
