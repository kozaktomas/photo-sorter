# MCP Photo Read and Similarity Tools for Book Workflow

Add read-only photo tools to the existing MCP server so AI agents can see photos and find similar ones when composing book layouts. These are NOT general photo management tools ‚Äî they exist specifically to support the book editing workflow.

## Prerequisites

The MCP server infrastructure exists in `internal/mcp/` with HTTP SSE transport and Bearer token auth.

## MCP Tools

1. **`get_photo`** ‚Äî Get photo metadata
   - Parameters: `photo_uid` (string, required)
   - Returns: photo metadata including:
     - `uid`, `title`, `description`, `taken_at` (date), `taken_at_local`
     - `width`, `height` (original dimensions)
     - `lat`, `lng` (GPS coordinates if available)
     - `camera_make`, `camera_model`, `lens_model`
     - `faces` ‚Äî list of detected faces with names (from subjects)
     - `labels` ‚Äî list of assigned labels
     - `favorite`, `private`, `type` (image/video/etc)
   - Use the existing PhotoPrism client methods (`GetPhoto`, `GetPhotoDetails`) and optionally enrich with face data from the local database

2. **`get_photo_thumbnail`** ‚Äî Get a photo thumbnail as base64-encoded image
   - Parameters: `photo_uid` (string, required), `size` (string, optional ‚Äî default `fit_720`, options: `fit_720`, `fit_1280`, `fit_2048`, `tile_500`, `tile_224`)
   - Returns: base64-encoded JPEG image data with mime type
   - Fetch the thumbnail via the existing PhotoPrism client (`GetPhotoDownload` or the thumbnail endpoint `/api/v1/photos/{uid}/thumb/{size}`), read the response body, and encode as base64
   - This allows multimodal AI agents (like Claude) to actually see the photo content
   - Include `width` and `height` of the returned thumbnail in the response

3. **`find_similar_photos`** ‚Äî Find visually similar photos using CLIP embeddings
   - Parameters: `photo_uid` (string, required), `limit` (int, optional ‚Äî default 10, max 50), `scope_section_id` (int, optional ‚Äî limit search to photos in a specific section), `scope_book_id` (int, optional ‚Äî limit search to photos in a specific book)
   - Returns: array of similar photos with:
     - `uid`, `title`, `description`, `taken_at`
     - `similarity` (float, 0-1, higher = more similar)
     - `in_book` (bool ‚Äî whether this photo is already placed on a page in the book)
     - `in_section` (string ‚Äî section title if assigned to one, empty otherwise)
   - Use the existing HNSW embedding index for fast similarity search
   - When `scope_section_id` or `scope_book_id` is provided, filter results to only include photos that belong to that section/book
   - The `in_book` / `in_section` enrichment helps the AI agent avoid suggesting photos that are already placed

## Implementation Notes

- Add tools in `internal/mcp/photos.go`
- Register tools in the existing server setup in `server.go`
- For `get_photo`: combine data from PhotoPrism API (`GetPhoto`) with local database (faces, embeddings status)
- For `get_photo_thumbnail`: the PhotoPrism thumbnail endpoint requires an authenticated session ‚Äî use the session created at server startup. Download the image bytes and base64-encode them. Return as `{ "mime_type": "image/jpeg", "data": "base64...", "width": N, "height": N }`
- For `find_similar_photos`: reuse the existing `HNSWEmbeddingIndex.Search()` method. The enrichment (in_book, in_section) requires querying the book database ‚Äî use `BookReader` to check if each result photo is assigned to pages/sections in the specified book
- Keep responses reasonably sized ‚Äî for `get_photo_thumbnail`, `fit_720` is a good default (small enough for base64 but large enough for AI to analyze content)
