# Fix Duplicate Uploads Not Added to Book Section

When uploading photos that already exist in the PhotoPrism library, they are correctly detected as duplicates but are NOT added to the selected book section. This is because `addToBookSection` only receives `newUIDs` (photos that didn't exist before).

## Root Cause

In `internal/web/handlers/upload_job.go`, the flow is:
1. `snapshotBefore` — captures album photo UIDs before upload
2. `uploadFiles` — uploads to PhotoPrism
3. `processUploadTokens` — PhotoPrism indexes the files
4. `detectNewPhotos` — diffs before/after to find only NEW photos
5. `applyPostUploadActions` — receives only `newUIDs`, calls `addToBookSection(ctx, job, newUIDs, result)`

If a photo already exists, PhotoPrism deduplicates it. It appears in the album (if not already there), but `detectNewPhotos` won't find it because it was already in `beforeUIDs`. So `newUIDs` is empty and `addToBookSection` gets nothing to add.

## Fix

`addToBookSection` should receive ALL photo UIDs from the album after upload, not just new ones. The book section should contain whatever the user uploaded, regardless of whether PhotoPrism considered it new or duplicate.

Approach:
1. After `processUploadTokens`, get the full list of album photo UIDs (the `afterUIDs` set)
2. Pass ALL uploaded photo UIDs (both new and existing) to `addToBookSection`
3. `AddSectionPhotos` in the database layer should already handle duplicates gracefully (ignore photos already in the section)

Alternative simpler approach:
- In `applyPostUploadActions`, when `BookSectionID` is set, always fetch the full album after processing and add ALL photos from the album diff (not just new ones). Or better: track which photos were in the upload batch (by matching filenames or using the album's full photo list after upload) and add those to the section.

The simplest fix: change `addToBookSection` to use the full `afterUIDs` set (all photos in album after upload) minus photos already in the book section, rather than just `newUIDs`.

## Files to modify

- `internal/web/handlers/upload_job.go` — pass all uploaded photos (not just new) to `addToBookSection`
