# Text Search on Texts Tab

Add a search input to the Texts tab to quickly find specific text content across the entire book.

## Requirements

### 1. Search input

A search box at the top of the Texts tab, above the text list:
- Placeholder: "Hledat v textech..." / "Search texts..."
- Filters the text list in real-time as the user types (client-side, no API call)
- Case-insensitive search
- Searches in: text content, source name (section/page name)
- Clear button (X) to reset the search
- Show result count: "3 z 24 textů" / "3 of 24 texts"

### 2. Highlighting

In the filtered results, highlight the matching substring within the text preview with a background color (e.g. `bg-yellow-500/30`).

### 3. Implementation

Pure frontend filtering — all text data is already loaded from the book detail response. Use a simple `includes()` or regex match on the normalized (lowercased) text.

### 4. i18n

Add keys:
- `books.editor.searchTexts`: "Hledat v textech..." (cs) / "Search texts..." (en)
- `books.editor.searchResults`: "{{count}} z {{total}} textů" (cs) / "{{count}} of {{total}} texts" (en)
