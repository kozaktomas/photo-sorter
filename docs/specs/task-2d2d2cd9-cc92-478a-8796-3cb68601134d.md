## Goal

Text in mixed layouts (`1p_2l`, `2l_1p`) is too close to adjacent photos. The text parbox fills the entire slot width with zero padding, leaving only the 4mm column gutter between text and photo. Add inner padding to text slots so text has breathing room.

## Problem

In `internal/latex/templates/book.tex` line 92-97, the text node uses `\parbox{ClipW mm}` — the full slot width. In a `1p_2l` layout, the text slot is ~87.65mm wide (cols 1-4) and the text goes edge-to-edge. The 4mm gutter to the photo column is not enough visual separation.

## Solution

Add **4mm horizontal padding** to text slots. This should only apply when the text slot is adjacent to a photo slot (mixed layouts), not when text is fullscreen.

### Option A (simpler, recommended): Reduce parbox width by padding amount

In the LaTeX template, shrink the text `\parbox` width and offset the node position:

```tex
% Current (book.tex line 92-97):
\node[anchor=north west,inner sep=0]
  at (ClipX, ClipY+ClipH)
  {\parbox{ClipW mm}{ ... }};

% New — add TextPadLeft/TextPadRight to TemplateSlot:
\node[anchor=north west,inner sep=0]
  at (ClipX + TextPadLeft, ClipY+ClipH)
  {\parbox{ClipW - TextPadLeft - TextPadRight mm}{ ... }};
```

### Backend changes: `internal/latex/latex.go`

Add fields to `TemplateSlot`:
```go
TextPadLeft  float64 // mm, padding from left edge of slot
TextPadRight float64 // mm, padding from right edge of slot
```

When building text slots, detect if the slot is in a mixed layout adjacent to photos:
- `1p_2l` slot 0 (text on left): add ~4mm right padding (away from photos)
- `2l_1p` slot 0 or 1 (text on left stacked): add ~4mm right padding
- If text is on the right side: add ~4mm left padding
- `1_fullscreen` or `2_portrait` with all text: no padding needed (or maybe small padding from page edge)

A simpler approach: always add 4mm padding on the side where photos are adjacent. Check if any sibling slot in the same page has a photo.

### Template changes: `internal/latex/templates/book.tex`

Update the text node to use padding values:
```tex
\node[anchor=north west,inner sep=0]
  at ({{printf "%.2f" (addFloat $slot.ClipX $slot.TextPadLeft)}},{{printf "%.2f" (addFloat $slot.ClipY $slot.ClipH)}})
  {\parbox{{print "{"}}{{printf "%.2f" (subFloat (subFloat $slot.ClipW $slot.TextPadLeft) $slot.TextPadRight)}}mm}{%
```

You'll need to add `subFloat` template function (or compute the width in Go and pass it as a field).

### Frontend changes

Update the web preview to match — text slots in `PagesTab.tsx` and `PreviewTab.tsx` should also show padding when adjacent to photos.
