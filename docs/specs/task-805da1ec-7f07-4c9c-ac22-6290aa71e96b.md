## Overview

Two album management improvements:
1. Photo detail page: show which albums a photo belongs to
2. Album detail page: select photos and remove them from the album

## Feature 1: Album membership on Photo Detail

### Backend

Create `GET /api/v1/photos/{uid}/albums` endpoint (similar pattern to existing `GET /api/v1/photos/{uid}/books`).

**Implementation approach:** PhotoPrism doesn't have a direct "get albums for photo" query. Two options:
- Option A: Use PhotoPrism search endpoint with photo UID filter and extract album info from response
- Option B: Fetch all albums (`GetAlbums()`), then for each check if photo is in it — too slow
- Option C (recommended): Use PhotoPrism's search API: `GET /api/v1/photos?uid={photoUID}` returns photo with album data, OR query albums with a search filter. Check `internal/photoprism/swagger.yaml` for available query parameters. Another approach: fetch all albums, then use `GetAlbumPhotos` with a filter — but this is N+1.

**Simplest approach:** Add a handler that fetches all user albums (`GetAlbums`), then for each album calls `GetAlbumPhotos` with count=1 and filters for the photo UID. This is N+1 but albums are typically few (10-50). Cache-friendly since album list is small.

**Better approach if available:** Check if PhotoPrism search supports `GET /api/v1/albums?photo={uid}` or similar. Check swagger.yaml.

Add handler in `internal/web/handlers/photos.go` or a new `photo_albums.go`. Register route in `internal/web/routes.go`.

Response format:
```json
[
  { "uid": "abc123", "title": "Album Name", "photo_count": 42 }
]
```

### Frontend

1. **API client** (`web/src/api/client.ts`): Add `getPhotoAlbumMemberships(uid)` function.

2. **AlbumMembership component** (`web/src/pages/PhotoDetail/AlbumMembership.tsx`): Similar to existing `BookMembership.tsx` (line 13-54). Show list of albums as clickable links (navigate to album detail). Display album title and photo count.

3. **Integrate in PhotoDetail** (`web/src/pages/PhotoDetail/index.tsx`): Add `AlbumMembership` next to existing `BookMembership`. Fetch album memberships in `usePhotoData` hook or a separate hook.

## Feature 2: Bulk remove photos from album detail

### Current state (already exists partially!)

- `web/src/pages/Photos/index.tsx` already has `usePhotoSelection` hook with bulk actions including "Remove from Album" (shown when `filters.selectedAlbum` is set)
- `web/src/pages/Albums.tsx` has `AlbumDetailPage` (line 115+) that shows photos in a `PhotoGrid` but does NOT use `usePhotoSelection`
- Backend `DELETE /api/v1/albums/{uid}/photos/batch` already exists
- Frontend `removePhotosFromAlbum()` already exists in `client.ts` (line 497)

### Implementation

The `AlbumDetailPage` in `web/src/pages/Albums.tsx` needs to support photo selection and bulk removal.

1. **Add selection mode to AlbumDetailPage**: Import and use `usePhotoSelection` hook (same as Photos page does). Pass `selectionMode`, `selectedPhotos`, `togglePhotoSelection` to `PhotoGrid`.

2. **Add BulkActionBar**: Show `BulkActionBar` when photos are selected. Configure it with a "Remove from Album" action that calls `removePhotosFromAlbum(albumUid, selectedPhotoUids)`.

3. **Refresh after removal**: After successful removal, refetch the album photos list. The `AlbumDetailPage` currently uses `useEffect` to load photos — refactor to a callback that can be re-invoked, or use a refresh counter.

4. **UI**: Add a toggle button (like on Photos page) to enter/exit selection mode. Show selected count and "Remove from Album" button in the bulk action bar.

Look at how `Photos/index.tsx` implements this (lines 36-54 for selection, lines 153-161 for BulkActionBar) and replicate the pattern in `Albums.tsx:AlbumDetailPage`.
