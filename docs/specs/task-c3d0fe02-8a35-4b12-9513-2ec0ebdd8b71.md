## Cíl

Text content v page slotech (sloty, které obsahují `text_content` místo `photo_uid`) musí být vždy zarovnán do bloku (text-align: justify).

## Kontext

V book editoru mohou page sloty obsahovat buď fotku (`photo_uid`) nebo text (`text_content`). Text se renderuje jako markdown (GFM) přes `marked.js` + DOMPurify. Aktuálně text není zarovnán do bloku — je potřeba přidat `text-align: justify` všude, kde se text slot renderuje.

## Kde opravit

### Frontend (web UI)
- `web/src/pages/BookEditor/PageSlot.tsx` — hlavní renderování slotů v editoru
- `web/src/pages/BookEditor/PreviewTab.tsx` / `PreviewModal.tsx` — náhled stránek
- Všude kde se renderuje HTML z `text_content` markdown slotu

Přidej `text-align: justify` na kontejner renderovaného textu v text slotech.

### PDF export (LaTeX)
- `internal/latex/` — šablony a generování LaTeX kódu pro text sloty
- Ujisti se, že LaTeX výstup pro textové sloty používá `\justifying` nebo je v prostředí, které zarovnává do bloku (LaTeX defaultně justifikuje, ale ověř, že to není přepsáno).

## Ověření

1. `./dev.sh` — rebuild a restart
2. V book editoru otevři stránku s textovým slotem — text musí být zarovnán do bloku
3. V preview — stejně tak
4. Export PDF — text v slotech zarovnán do bloku
5. `cd web && npx tsc --noEmit && npm run lint` — frontend checks
6. `make lint` — Go checks (pokud se mění Go kód)
