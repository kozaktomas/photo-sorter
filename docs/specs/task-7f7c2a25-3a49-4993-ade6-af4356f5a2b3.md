# PDF Export: Photo Quality Options

Add a quality tier selector to PDF book export so users can choose between preview-quality and print-ready output.

## Requirements

- Add a `photo_quality` parameter to PDF export (both sync `GET /api/v1/books/{id}/export-pdf` and async `POST /api/v1/books/{id}/export-pdf/job`). Values: `low`, `medium`, `original`. Default: `medium` (preserves current behavior).
- `low`: download `fit_720` thumbnails. For quick previews only; PDF should be small and fast.
- `medium`: download `fit_3840` thumbnails (current behavior — no change).
- `original`: download full originals via `GetPhotoDownload` for final print. Handle HEIC/RAW by converting to JPEG before passing to lualatex. Cap the longest side at a sensible maximum (e.g. 8000 px) — resize anything larger. Document the cap in `docs/photo-book.md`.
- Expose the option in the web book editor export flow (dropdown next to export button) and in the MCP `update_book` / export flow if applicable — otherwise leave MCP at `medium` default for now.
- Single-page preview (`GET /api/v1/pages/{id}/export-pdf`) keeps current `medium` behavior, no new parameter.
- Preflight should warn when `photo_quality=original` is combined with photos that have a download-size smaller than what `medium` already gave them (i.e. "original" is lower quality than "medium" for that file).
- Update `docs/photo-book.md`, `docs/API.md`, and CLAUDE.md sections that describe PDF export to document the new parameter and the longest-side cap.

## Implementation Notes

- Photo fetch happens in `internal/latex/latex.go` `downloadPhoto` / `downloadPhotosWithProgress`. Plumb quality through `ExportOptions`.
- PhotoPrism thumbnail sizes: `fit_720`, `fit_1920`, `fit_2560`, `fit_3840`, `fit_7680`. Originals via `pp.GetPhotoDownload(hash)`.
- Use `image` stdlib + `golang.org/x/image` (or existing deps) for HEIC→JPEG and resize-to-cap.
- Expect PDF sizes with `original` to be multi-GB for large books — make sure the SSE progress phase `downloading_photos` reports byte totals correctly and the temp file cleanup still works.
