## Goal

When a chapter has a color (e.g. dark red), H1 headings render with a colored background box. The box is technically `\linewidth` wide (full parbox width), but visually appears narrower than the body text below it because the heading text has `\hspace{12pt}` internal padding on both sides. This creates an optical illusion of misalignment.

## Problem

In `internal/latex/markdown.go` line 175-183, the colored H1 is rendered as:

```latex
\definecolor{chaptercolor}{HTML}{8B0000}
{\fboxsep=0pt\noindent\colorbox{chaptercolor}{\parbox{\linewidth}{
  \vspace{10pt}\hspace{12pt}\fontsize{18}{22}\selectfont\sffamily\bfseries\textcolor{white}{HEADING TEXT}\hspace{12pt}\vspace{10pt}
}}}
\par\vspace{4mm}
```

The `\colorbox` + `\parbox{\linewidth}` makes the box full width, which is correct. But the `\hspace{12pt}` inside creates ~4mm padding for the heading text, while body text below starts at x=0. This makes the heading text look offset/narrower compared to body text.

## Solution

Make the colored heading box extend **beyond** the parbox by a negative margin, so the colored background bleeds to match or exceed the text width. OR, simpler: add the same left indent to body text paragraphs.

### Recommended approach: extend the colorbox with negative margins

Replace the current colorbox with one that uses `\makebox` to extend beyond `\linewidth`:

```latex
% Option: extend box 4mm on each side (matching the text padding visual feel)
{\fboxsep=0pt\noindent\hspace{-4mm}\colorbox{chaptercolor}{\parbox{\dimexpr\linewidth+8mm}{
  \vspace{10pt}\hspace{16pt}...\hspace{16pt}\vspace{10pt}
}}}
```

This makes the colored box visually wider than the text column, creating a "bleed" effect that looks intentional and professional rather than misaligned.

### Alternative approach: match internal padding

Keep the box at `\linewidth` but reduce internal padding to 6pt (from 12pt) so the visual offset is smaller:

```latex
\vspace{8pt}\hspace{6pt}...\hspace{6pt}\vspace{8pt}
```
