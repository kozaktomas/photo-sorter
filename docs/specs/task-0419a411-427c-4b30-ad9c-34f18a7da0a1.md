## Goal

In the book text editor/preview, `~` is used as a non-breaking space (Czech typography convention). But when two tildes appear near each other (e.g. `slovo~ ~slovo` or `~~text~~`), GFM markdown interprets `~~text~~` as strikethrough (`<del>text</del>`) before our custom `~` → `&nbsp;` replacement runs.

## Problem

In `web/src/utils/markdown.ts`:

1. Line 104: `marked.parse()` runs with `gfm: true` — this enables GFM strikethrough (`~~text~~` → `<del>`)
2. Line 91: `html.replace(/~/g, '&nbsp;')` runs AFTER marked.js — too late, the `~~` pairs are already consumed

So any text with adjacent tildes gets corrupted into strikethrough instead of non-breaking spaces.

## Solution

**Escape tildes before passing to marked.js**, then restore them after.

In `web/src/utils/markdown.ts`, add a pre-processing step in `renderMarkdown()` before `marked.parse()`:

```typescript
// Before marked.parse: protect tildes from GFM strikethrough
// Replace \~ (escaped tilde) with placeholder first
const ESCAPED_TILDE = '\x00ESC_TILDE\x00';
const TILDE = '\x00TILDE\x00';
let preprocessed = md.replace(/\\~/g, ESCAPED_TILDE);
preprocessed = preprocessed.replace(/~/g, TILDE);

// Parse markdown (no tildes left, so GFM strikethrough won't trigger)
const html = marked.parse(processAlignmentMacros(stripTableWidthHints(preprocessed))) as string;

// After marked: restore tildes as &nbsp; (or literal ~ for escaped)
html = html.replaceAll(TILDE, '&nbsp;');
html = html.replaceAll(ESCAPED_TILDE, '~');
```

Then simplify `processInlineMacros()` — remove the tilde handling from there since it's now done in the main flow.

**Alternative simpler approach:** Disable GFM strikethrough in marked config. We don't use strikethrough anywhere in book texts:

```typescript
const marked = new Marked({
  breaks: true,
  gfm: true,
  extensions: [{
    name: 'del',
    level: 'inline',
    start(src) { return undefined; }, // disable
    tokenizer() { return undefined; },
  }],
});
```

Either approach works. The pre-processing approach is more robust. The extension disable is simpler but might not work cleanly with all marked versions.
