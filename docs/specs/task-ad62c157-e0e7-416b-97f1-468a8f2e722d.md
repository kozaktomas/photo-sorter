# Add Source Sans as Secondary Font for Headings

Add Source Sans 3 (formerly Source Sans Pro) as a secondary sans-serif font used for headings in colored chapter boxes. EB Garamond remains the primary body text font.

## Requirements

### Frontend

- Add Source Sans 3 as a Google Web Font in `web/index.html` (weights: 400, 600, 700)
- Update the typography config (`web/src/constants/bookTypography.ts`) to include Source Sans as the heading font:
  - `h1.fontFamily: "'Source Sans 3', sans-serif"`
  - Body text remains EB Garamond
- The WYSIWYG preview should use Source Sans for H1 headings (and potentially H2 if desired)

### LaTeX

- Add Source Sans 3 as a sans-serif font in `internal/latex/templates/book.tex` using fontspec:
  ```latex
  \setsansfont{SourceSans3}[
    UprightFont = *-Regular,
    BoldFont = *-SemiBold,
    ItalicFont = *-Italic,
    BoldItalicFont = *-SemiBoldItalic,
  ]
  ```
- Verify that Source Sans 3 is available on the system (it's included in `texlive-fonts-extra` or can be installed via the OS package manager). If not available, add a note about installation.
- H1 rendering in LaTeX should use `\textsf{...}` to switch to the sans-serif font

### Docker

- Ensure the font is available in the Docker build environment if PDF export runs in Docker. Check if `texlive-fonts-extra` is already included in the Dockerfile, or if the font needs to be added separately.

## What NOT to Do

- Do not replace EB Garamond — it stays as the main serif font for body text
- Do not change font sizes — only add the new font family
- Do not add more than the needed weights (400, 600, 700) to keep page load fast
