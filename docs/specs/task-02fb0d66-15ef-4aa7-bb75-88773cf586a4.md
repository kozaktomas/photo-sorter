## Summary

Remove automatic insertion of section/chapter titles into the PDF export. Users will always place titles manually using text slots (text instead of photo in a page slot).

## Current Behavior

In `internal/latex/latex.go`, the `buildContentPage` function detects when a page is the first page of a titled section (`pageIndexInSection == 0 && sectionTitle != ""`). When true, it:
1. Reserves vertical space (`SectionHeadingHeightMM`) at the top of the canvas
2. Renders a decorative rule line and the section title text in the LaTeX template (`book.tex`, lines 57-68)
3. Reduces effective canvas height for photo/text slots on that page

The LaTeX template (`internal/latex/templates/book.tex`) renders the section heading block with `HasSectionTitle`, `SectionRuleY`, `SectionTitleY`, and `SectionTitle` fields.

## Required Changes

### LaTeX Template (`internal/latex/templates/book.tex`)

- Remove the `{{- if $page.HasSectionTitle}}...{{- end}}` block (lines 57-68) that renders the inline section heading (rule line + title text)

### Go Code (`internal/latex/latex.go`)

- In `buildContentPage`: remove the section heading space computation (the `hasSectionTitle` block that adjusts `effectiveCanvasTopY` and `slotConfig.CanvasHeightMM`)
- Remove `sectionTitle string` and `pageIndexInSection int` parameters from `buildContentPage` since they are no longer needed
- Update the call site in `buildSection` accordingly (currently passes `g.title` and loop index `i`)
- Remove unused fields from `TemplatePage` struct: `HasSectionTitle`, `SectionTitle`, `SectionTitleY`, `SectionRuleY`, `RunningLeft`, `RunningRight`
- Clean up `runningLeft`/`runningRight` computation block (currently unused in the template but still computed)
- Use `canvasTopY` directly instead of `effectiveCanvasTopY` (which was only different when section title was present)

### Tests (`internal/latex/latex_test.go`)

- Update any tests that assert on `HasSectionTitle`, `SectionTitle`, `SectionTitleY`, `SectionRuleY`, `RunningLeft`, or `RunningRight` fields
- Verify that pages previously getting reduced canvas height now use full canvas height

### Cleanup

- Remove `SectionHeadingHeightMM` constant if it becomes unused
- Run `make check` to ensure everything compiles and tests pass
