## Bug

Při změně barvy kapitoly se smaže její název. Frontend posílá `updateChapter(id, { color })` bez `title`, backend to zapíše celé a title se přepíše na prázdný string.

## Příčina

Dva problémy:

### 1. Frontend posílá jen měněné pole
`web/src/pages/BookEditor/SectionSidebar.tsx:386`:
```js
await updateChapter(id, { color });  // chybí title
```
Řádek 379 má podobný problém opačně:
```js
await updateChapter(id, { title });  // chybí color
```

### 2. Backend nepodporuje partial update
`internal/web/handlers/books.go:396`:
```go
chapter := &database.BookChapter{ID: id, Title: req.Title, Color: req.Color}
```
Vždy přepíše oba fieldy, i když přišel jen jeden.

## Oprava

Buď:

**A) Backend partial update (preferovaný)** — načíst aktuální chapter z DB, aplikovat jen neprázdné fieldy z requestu, a pak uložit. Pozor: prázdný title by měl být validní stav? Spíš ne — title by měl být povinný, color volitelný.

**B) Frontend vždy posílá oba fieldy** — při změně barvy poslat i aktuální title a naopak. Jednodušší, ale křehčí.

Doporučuji variantu A: v handleru `UpdateChapter` (`internal/web/handlers/books.go:382`) nejdřív načíst chapter z DB (přes BookReader), pak přepsat jen pole, která přišla v requestu (použít pointer typy nebo kontrolu na nenulové hodnoty).

## Soubory k úpravě

- `internal/web/handlers/books.go` — handler `UpdateChapter` (~řádek 382)
- Případně `internal/database/postgres/books.go` — pokud je potřeba přidat metodu `GetChapter`
- `web/src/pages/BookEditor/SectionSidebar.tsx` — řádky 379 a 386, jako zálohu poslat oba fieldy

## Ověření

1. Nastavit kapitole název "Test" → ověřit že se uloží
2. Změnit barvu kapitoly → název musí zůstat "Test"
3. Přejmenovat kapitolu → barva musí zůstat
4. `make lint && make test`
5. `cd web && npx tsc --noEmit && npm run lint`
