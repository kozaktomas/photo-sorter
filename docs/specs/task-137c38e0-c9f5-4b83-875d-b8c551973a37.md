## Cíl

Přidat podporu vlastních maker pro zarovnání odstavců v textových slotech fotoknihy:

- `->text<-` — zarovnání na střed (center)
- `->text->` — zarovnání doprava (right)
- Bez makra — výchozí zarovnání (justify, resp. left)

Makra fungují na úrovni celého odstavce (řádku).

## Příklady

```
Normální odstavec zarovnaný do bloku.

->Tento text je vycentrovaný.<-

->Tento text je zarovnaný doprava.->
```

## Kde implementovat

### 1. LaTeX konvertor
`internal/latex/markdown.go` — funkce `markdownToLatexInternal`:
- Detekovat pattern `->...<-` a `->...->` na začátku/konci trimnutého řádku
- `->text<-` → `\begin{center}text\end{center}` nebo `{\centering text\par}`
- `->text->` → `{\raggedleft text\par}`
- Aplikovat před/po ostatních inline transformacích (bold, italic, czech typography)

### 2. Frontend markdown renderer
`web/src/utils/markdown.ts` — renderování markdown do HTML:
- `->text<-` → `<p style="text-align: center">text</p>`
- `->text->` → `<p style="text-align: right">text</p>`
- Aplikovat jako pre-processing krok před `marked.js` parsováním, nebo jako post-processing na HTML

### 3. Frontend WYSIWYG preview
Pokud existuje WYSIWYG preview textových slotů, zarovnání musí být viditelné i tam.

## Důležité

- Makra musí fungovat i v kombinaci s inline formátováním: `->**tučný text**<-` → tučný vycentrovaný
- Makra jsou na úrovni řádku/odstavce, ne inline — celý řádek je zarovnaný
- V rozpracované knize nejsou žádná taková makra, takže zpětná kompatibilita není problém

## Ověření

1. `./dev.sh` — rebuild
2. V book editoru vytvořit textový slot s makry → ověřit ve frontend preview
3. Export PDF → ověřit zarovnání v LaTeX výstupu
4. Kombinace s bold/italic/nadpisy
5. `make lint && make test`
6. `cd web && npx tsc --noEmit && npm run lint`
