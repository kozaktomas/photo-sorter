## Problem

When exporting a single page PDF preview (`GET /api/v1/pages/{id}/export-pdf`), the page is always rendered as page 1 (recto/right-hand side). This means:
- Page number is always "1" instead of the actual page number in the book
- Margins are always recto (inside=left, outside=right) even if the page would be on the left (verso) in the final book
- Folio anchor is always "south east" even for verso pages
- Caption markers are always placed as recto

The user wants single page preview to match the final book render exactly — correct page number, correct left/right side, correct margins.

## Root Cause

In `internal/latex/latex.go`, `GenerateSinglePagePDF` (line ~1111) hardcodes:
```go
pb := &pageBuilder{
    totalContentPages: 1,
    contentPageIdx:    0,
    pageNumber:        0,  // becomes 1 after increment → always recto
}
```

## Solution

Calculate the actual page number by determining the page's position in the full book page sequence, then pass it to the single page renderer.

### Step 1: Backend handler — compute actual page number

In `internal/web/handlers/books.go`, `ExportPagePDF` handler (~line 1828):

1. After fetching the page and its book, also fetch ALL pages and sections for the book (same as full export does)
2. Sort pages using `sortPagesBySectionOrder()` (already exists in latex.go)
3. Find the target page's index in the sorted list
4. Compute the actual page number: `pageNumber = index + 1` (or `index + 2` if the book has a title, since title page = page 1)

### Step 2: Extend `SinglePageInput` struct

In `internal/latex/latex.go`, add `PageNumber` field to `SinglePageInput`:

```go
type SinglePageInput struct {
    Page         database.BookPage
    BookTitle    string
    ChapterColor string
    Captions     CaptionMap
    PageNumber   int  // NEW: actual page number in book (1-based)
}
```

### Step 3: Update `GenerateSinglePagePDF`

In `internal/latex/latex.go`, `GenerateSinglePagePDF` (~line 1111):

Use `input.PageNumber` instead of hardcoding:
```go
pb := &pageBuilder{
    config:            config,
    photos:            photos,
    captions:          input.Captions,
    totalContentPages: input.PageNumber, // at least this many
    contentPageIdx:    input.PageNumber - 1,
    pageNumber:        input.PageNumber - 1, // will be incremented to PageNumber
    photoSet:          make(map[string]bool),
}
pb.contentPageIdx++ // not needed if we set it directly
pb.pageNumber++
```

Or simpler — set `pageNumber` directly to `input.PageNumber - 1` so after the `pb.pageNumber++` it becomes the correct number.

This will make `isRecto = pageNumber % 2 == 1` evaluate correctly, and all margins, folio placement, and caption markers will match the full book render.

### Step 4: Handler passes page number

```go
input := latex.SinglePageInput{
    Page:         *page,
    BookTitle:    book.Title,
    ChapterColor: chapterColor,
    Captions:     captions,
    PageNumber:   actualPageNumber, // computed in step 1
}
```

### Edge cases

- Book with no title: first content page = page 1 (recto)
- Book with title: title page = page 1, first content page = page 2 (verso)
- If somehow page is not found in the sorted list (shouldn't happen), fall back to pageNumber=1
