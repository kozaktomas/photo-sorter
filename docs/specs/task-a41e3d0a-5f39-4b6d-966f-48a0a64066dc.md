# Automatic HTML-to-Markdown Paste Conversion

When users paste content from Word, Google Docs, or LibreOffice Writer into text editors in the book editor, the formatted HTML should be automatically converted to clean Markdown.

## Requirements

- Install `turndown` and `turndown-plugin-gfm` npm packages (plus `@types/turndown` for TypeScript)
- Intercept `onPaste` events on all textareas used for text editing in the book editor
- Read `text/html` from the clipboard via `event.clipboardData.getData('text/html')`
- If HTML is present, convert it to Markdown using `turndown` and insert the result instead of the raw paste
- If no HTML is present in the clipboard, fall back to default paste behavior (plain text)
- Configure turndown to produce GFM-compatible Markdown (headings, bold, italic, lists, tables via the GFM plugin)
- Strip unnecessary styling artifacts from Word/Google Docs (turndown handles this by default — it ignores inline styles, font colors, etc.)

## Affected Components

Two components have textareas that need the paste handler:

1. **TextSlotDialog** in `web/src/pages/BookEditor/PagesTab.tsx` — the textarea for editing text slot content (around line 281)
2. **PhotoDescriptionDialog** in `web/src/pages/BookEditor/PhotoDescriptionDialog.tsx` — the textarea for photo descriptions (around line 234) and notes (around line 397)

## Implementation Notes

- Create a shared utility function (e.g. in `web/src/utils/clipboard.ts` which already exists, or a new `web/src/utils/paste.ts`) that handles the paste conversion logic
- The utility should create a single `TurndownService` instance (not per-paste) with GFM plugin enabled
- The paste handler should: prevent default, get HTML from clipboard, convert to Markdown, insert at cursor position respecting existing text (before cursor + markdown + after cursor), and update the React state
- If the textarea has selected text when pasting, the paste should replace the selection (standard behavior)
- Turndown rule configuration: `headingStyle: 'atx'`, `bulletListMarker: '-'`, `codeBlockStyle: 'fenced'`
- Make sure the word/character count updates after paste (it should, since we update the state value)

## Edge Cases

- Pasting from a source that provides both `text/html` and `text/plain` (e.g. copying from a browser): prefer HTML conversion since it preserves formatting
- Pasting plain text only (e.g. from terminal): no HTML in clipboard, fall through to normal behavior
- Pasting an image or file: ignore, let default behavior handle it
- Very large paste (e.g. entire Word document): should still work, turndown is fast enough for any reasonable document size
