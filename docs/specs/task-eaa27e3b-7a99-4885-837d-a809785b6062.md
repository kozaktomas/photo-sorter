# Texts Tab in Book Editor

Add a new "Texty" tab to the Book Editor that provides an overview of all text content in the book (both photo descriptions from sections and text slots from pages) with stats, AI operations, and navigation.

## Requirements

### 1. New tab

Add a "Texty" (Texts) tab to the Book Editor tab bar, alongside existing tabs (Sekce, Stránky, Náhled, Duplikáty).

### 2. Stats panel at the top

Display three statistics cards at the top of the tab:
- **Počet textů** — total count of all texts (photo descriptions + text slot contents)
- **Zkontrolované** — count of texts that have been AI-checked (no errors found or corrections accepted)
- **S chybou** — count of texts where AI check found errors that haven't been accepted yet

For tracking check status: use in-memory state within the tab session. When a user runs "Kontrola textu" on a text and the result comes back, mark that text as checked (clean or has errors). This state resets when leaving the tab, which is acceptable.

### 3. Text list

Below the stats, show a list/table of all texts in the book. Each row shows:
- **Text preview** — first ~100 characters of the text, truncated
- **Source** — where the text is used: section name + photo (for descriptions) or page number + slot (for text slots)
- **Status** — unchecked / clean / has errors (visual indicator: gray / green / red dot or badge)
- **Actions:**
  - **Kontrola textu** button ($ icon) — runs AI text check inline, shows result expandable below the row
  - **Upravit délku** button ($ icon) with length selector — runs AI rewrite
  - **Přijmout** button on results to accept changes (saves via API)
  - **Přejít na stránku** link — navigates to the Pages tab with the relevant page selected

### 4. Data loading

To populate the list, use existing API data:
- Photo descriptions: from sections data (already loaded in book editor via `GET /api/v1/books/{id}`)
- Text slot contents: from pages data (slots with `text_content`)

No new backend endpoints should be needed — combine data from the existing book detail response which includes sections, pages, and slots.

### 5. AI operations

Reuse the existing `checkText` and `rewriteText` API client functions. After accepting a correction:
- For photo descriptions: call `updateSectionPhoto` to save
- For text slots: call the page slot update endpoint (`PUT /api/v1/pages/{id}/slots/{index}`) with updated `text_content`

### 6. Navigation

The "Přejít na stránku" link should:
- For text slots: switch to the Pages tab and select the page containing the slot
- For photo descriptions: switch to the Sections tab and select the section containing the photo

This requires the Book Editor to support programmatic tab switching. Add a callback or shared state that child tabs can use to trigger navigation.
