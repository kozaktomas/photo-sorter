# Sections Tab: Modal Description Editing + 3-Column Grid

Refactor the Book Editor's Sections tab to improve photo editing UX and layout consistency with the Pages tab.

## Requirements

### 1. Replace inline editing with modal dialog

Currently, `SectionPhotoPool.tsx` uses inline click-to-edit textareas below each photo for description and note fields. Replace this with the existing `PhotoDescriptionDialog` modal component, matching the Pages tab behavior.

**Current behavior (Sections tab):**
- Click on description/note text below photo → inline textarea appears
- Save on blur or Shift+Enter
- All editing state managed locally in `SectionPhotoPool.tsx` (`editingDesc`, `editingNote`, `descText`, `noteText`)

**Desired behavior (matching Pages tab):**
- Click on a photo (or an edit button) → `PhotoDescriptionDialog` modal opens
- Modal shows photo thumbnail preview, description textarea, and note textarea
- Explicit Save button to confirm changes
- Close button to cancel

**Implementation approach:**
- Import and reuse the existing `PhotoDescriptionDialog` component from `web/src/pages/BookEditor/PhotoDescriptionDialog.tsx`
- Remove the inline editing state and handlers (`editingDesc`, `editingNote`, `descText`, `noteText`, `handleDescBlur`, `handleNoteBlur`, etc.) from `SectionPhotoPool.tsx`
- Add state for `editingPhoto` (photo UID) and `editingPhotoData` (description + note) to trigger the modal
- Wire up the modal's `onSaved` callback to reload section photos

### 2. Change photo grid to 3 columns

Currently, `SectionPhotoPool.tsx` uses a responsive grid:
```
grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4
```

Change to a fixed 3-column grid:
```
grid grid-cols-3
```

This makes photos larger and easier to review. Keep the existing `gap-3` spacing.

## Files to modify

- `web/src/pages/BookEditor/SectionPhotoPool.tsx` — main changes (remove inline editing, add modal, change grid)

## Edge Cases

- If a photo has no description or note yet, the modal should show empty textareas (existing behavior in PhotoDescriptionDialog)
- After saving in the modal, the photo list should refresh to show updated text
- The inline description/note preview text below photos can remain as read-only display (truncated) — just remove the click-to-edit behavior
