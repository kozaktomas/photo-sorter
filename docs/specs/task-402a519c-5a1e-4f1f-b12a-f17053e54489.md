# MCP Server for Photo Book Management — Infrastructure + Book/Chapter Tools

Add an MCP (Model Context Protocol) server to the photo-sorter binary that exposes photo book management operations as MCP tools. This enables AI agents (Claude Code, Claude Desktop) to programmatically create and manage photo books.

## Requirements

### CLI Command

- Add a new Cobra command: `go run . mcp-serve`
- Starts an MCP server using HTTP SSE transport
- Flags:
  - `--port` (default: 8086)
  - `--host` (default: 0.0.0.0)
- The server needs access to the database (for book operations) and PhotoPrism client (for photo metadata/thumbnails). Reuse existing config loading from environment variables.

### MCP SDK

- Use the `github.com/mark3labs/mcp-go` Go library (the most popular Go MCP SDK)
- Configure HTTP SSE transport (not stdio)
- Server name: "photo-sorter-books"
- Server version: use the existing version injection from build

### Authentication

- API token-based auth via environment variable `MCP_API_TOKEN`
- MCP clients pass the token as `Authorization: Bearer <token>` in HTTP headers
- If `MCP_API_TOKEN` is not set, the server should refuse to start (log error and exit)
- The server authenticates to PhotoPrism internally using the configured `PHOTOPRISM_USERNAME` / `PHOTOPRISM_PASSWORD` credentials (same as the web server does)
- Create a single PhotoPrism session at startup, reuse it for all MCP tool calls

### MCP Tools — Books

Implement these MCP tools as thin wrappers over existing database/API operations:

1. **`list_books`** — List all photo books
   - No parameters
   - Returns: array of books with id, title, description, created_at

2. **`get_book`** — Get book detail with chapters, sections, pages
   - Parameters: `book_id` (int, required)
   - Returns: full book structure (chapters, sections, section photo counts, pages with slots)

3. **`create_book`** — Create a new book
   - Parameters: `title` (string, required), `description` (string, optional)
   - Returns: created book

4. **`update_book`** — Update book title/description
   - Parameters: `book_id` (int, required), `title` (string, optional), `description` (string, optional)
   - Returns: updated book

5. **`delete_book`** — Delete a book and all its content
   - Parameters: `book_id` (int, required)
   - Returns: success confirmation

### MCP Tools — Chapters

6. **`create_chapter`** — Create a chapter in a book
   - Parameters: `book_id` (int, required), `title` (string, required), `color` (string, optional — hex color like `#8B0000`)
   - Returns: created chapter

7. **`update_chapter`** — Update chapter title/color
   - Parameters: `chapter_id` (int, required), `title` (string, optional), `color` (string, optional)
   - Returns: updated chapter

8. **`delete_chapter`** — Delete a chapter
   - Parameters: `chapter_id` (int, required)
   - Returns: success confirmation

9. **`reorder_chapters`** — Reorder chapters in a book
   - Parameters: `book_id` (int, required), `chapter_ids` (array of ints, required — new order)
   - Returns: success confirmation

### Implementation Notes

- Reuse existing database repository methods (`BookReader`, `BookWriter` interfaces) — do NOT call the HTTP API endpoints, call the Go functions directly
- For operations that need PhotoPrism data (e.g. photo thumbnails in book detail), use the PhotoPrism client created at startup
- Each MCP tool handler should validate parameters and return clear error messages
- Use structured JSON responses that are easy for AI to parse

### Package Structure

```
internal/mcp/
├── server.go          # MCP server setup, auth middleware, lifecycle
├── books.go           # Book and chapter tool handlers
```

The `cmd/` directory gets a new `mcp_serve.go` file with the Cobra command.

## What NOT to Do

- Do not implement stdio transport — HTTP SSE only
- Do not add tools for albums, labels, photo search, or anything outside photo book management
- Do not add preflight or auto-layout tools
- Do not create a separate binary — this is a new command in the existing binary
