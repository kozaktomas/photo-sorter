# Per-Chapter Color Theme with Colored H1 Headings

Each book chapter should have its own accent color. H1 headings in text slots render as full-width colored boxes with white text in a sans-serif font, visually distinguishing chapters from each other.

## Requirements

### Database

- Add a `color` column (VARCHAR, nullable) to the `book_chapters` table via a new migration. Stores a hex color string (e.g. `#8B0000`). NULL means no color / default.

### Backend API

- The existing `PUT /api/v1/chapters/{id}` endpoint already updates chapter properties. Extend it to accept and persist the `color` field.
- The `GET /api/v1/books/{id}` response already includes chapters — ensure the `color` field is included in the chapter objects.

### Frontend — Chapter Color Picker

- In the BookEditor, wherever chapters are managed (chapter list, chapter edit UI), add a color input for each chapter.
- Use a standard HTML `<input type="color">` or a simple text input for hex values. No need for a fancy color picker library.
- Show a color swatch next to the chapter name in the chapter list for quick visual reference.

### Frontend — WYSIWYG Preview Integration

- This task depends on the WYSIWYG preview task (typography config). The `BOOK_TYPOGRAPHY.h1` config defines default H1 styling.
- When rendering a text slot that belongs to a section within a chapter that has a color set, override the H1 styling:
  - `backgroundColor`: the chapter's color
  - `color`: `#ffffff` (white text)
  - `fontFamily`: sans-serif (e.g. system sans-serif stack: `'Inter', 'Segoe UI', system-ui, sans-serif`)
  - `padding`: appropriate padding for a box look (e.g. `4px 12px`)
  - `width`: 100% (full width of the text slot)
- If the chapter has no color set, H1 renders with default typography config styling.

### Frontend — Visual Chapter Indicators

- In the BookEditor sections/pages list, show the chapter color as a subtle accent (e.g. a colored left border or dot next to sections belonging to that chapter).
- This helps the user see the chapter structure at a glance.

### LaTeX / PDF Export

- When exporting to PDF, if a chapter has a color set, render H1 headings using `\colorbox` or `tcolorbox` with:
  - Background: the chapter's color
  - Text: white (`\textcolor{white}{...}`)
  - Font: sans-serif (add a sans-serif font to the LaTeX template, e.g. `\setsansfont{...}` via fontspec, and use `\textsf{...}` for H1)
  - Full width of the text parbox
- If no color is set, render H1 as before (current default styling).
- The chapter color hex value needs to be converted to a LaTeX color definition (e.g. `\definecolor{chaptercolor}{HTML}{8B0000}`).

## Data Flow

1. User sets chapter color in BookEditor UI
2. `PUT /api/v1/chapters/{id}` saves color to DB
3. Book data reload includes chapter colors
4. TextSlotDialog WYSIWYG preview reads chapter color from book data, applies to H1 rendering
5. PDF export reads chapter color from DB, applies to LaTeX H1 rendering

## Edge Cases

- Section not assigned to any chapter: use default H1 styling (no colored box)
- Chapter with no color set: use default H1 styling
- Text slot with no H1 headings: chapter color has no visible effect (that's fine)
- Multiple H1 headings in one text slot: all get the chapter color
- Color accessibility: this is the user's choice, no need to validate contrast ratios

## What NOT to Do

- Do not add a predefined color palette — the user picks any color via hex input or color picker
- Do not add sans-serif font as a Google Web Font for now — use system sans-serif stack in CSS. For LaTeX, pick a commonly available sans-serif (e.g. the already-available Latin Modern Sans or add a note about font choice)
