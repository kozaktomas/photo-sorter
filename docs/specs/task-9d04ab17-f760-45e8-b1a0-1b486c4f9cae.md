## Problem

When a user selects a dark chapter background color in the Book Editor, the H1 heading text remains dark and becomes unreadable against the dark background. This affects both the page preview in the UI and the exported PDF.

## Solution

Automatically determine whether to use white or dark text for H1 headings based on the chapter's background color luminance.

### Implementation

1. **Luminance calculation utility** — Create a helper function that computes relative luminance from a hex color using the WCAG 2.0 formula:
   - Convert hex to RGB (0-255)
   - Linearize each channel: `c <= 0.03928 ? c/12.92 : ((c+0.055)/1.055)^2.4`
   - `L = 0.2126*R + 0.7152*G + 0.0722*B`
   - If `L < 0.5` (dark background) → use white text; otherwise → use dark text

2. **Frontend (page preview + Preview tab)** — Where chapter color is applied to H1 headings, also compute the text color and apply it. Check:
   - `PageTemplate.tsx` or wherever H1 gets the chapter color as background
   - `PreviewTab.tsx` for the preview rendering
   - Any inline styles or CSS classes that set H1 color

3. **PDF export (LaTeX)** — In the LaTeX template for H1 headings (`internal/latex/`), when the chapter color is applied, also compute and set the text color. The luminance calculation needs to exist in Go as well (or passed as a parameter from the layout logic).

### Scope

- Both frontend preview AND PDF export must respect this
- The threshold (0.5) should be a constant, easy to tweak
- No new UI controls needed — this is fully automatic
- Test with edge cases: pure black (#000000), pure white (#FFFFFF), mid-grays, saturated dark colors (dark blue, dark red)
