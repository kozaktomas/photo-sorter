## Problem

Photo captions (footer text on book pages) are currently plain text — no inline formatting support. Users need bold (`**text**`), italic (`*text*`), and non-breaking spaces (`~`) in captions.

## Current state

Caption text flows through `latexEscape()` which escapes ALL LaTeX special characters including `~` → `\textasciitilde{}`, `*` → kept as literal `*`. No markdown/formatting is applied to captions — only to text slots (which use `MarkdownToLatex()`).

Key files:
- `internal/latex/latex.go` — `latexEscape()` (line ~938), `latexEscapeRaw()` (line ~912), `writeFooterCaption()` (line ~993)
- `internal/latex/markdown.go` — has inline formatting regexes for `**bold**` and `*italic*` already used by text slots

## Goal

Add lightweight inline formatting to captions:
1. `**text**` → `\textbf{text}` (bold)
2. `*text*` → `\textit{text}` (italic)
3. `~` → `~` (LaTeX non-breaking space, i.e. do NOT escape it to `\textasciitilde{}`)

Captions should NOT support full markdown (no headings, lists, blockquotes, tables). Only these 3 inline features.

## Implementation

Create a new function (e.g., `latexEscapeCaption()`) or modify the caption rendering path:

1. **Non-breaking space:** In `latexEscapeRaw()`, the `~` is escaped to `\textasciitilde{}`. For captions, `~` should remain as `~` (which is LaTeX's non-breaking space). Either:
   - Create a `latexEscapeCaptionRaw()` variant that omits the `~` → `\textasciitilde{}` replacement, OR
   - Process captions with a separate function that first extracts `~`, escapes everything else, then restores `~`

2. **Bold and italic:** After LaTeX-escaping the caption text, apply the same inline formatting regexes from `markdown.go`:
   - `\*\*(.+?)\*\*` → `\textbf{$1}`
   - `\*(.+?)\*` → `\textit{$1}`
   
   Note: `*` is not a LaTeX special character, so it survives `latexEscapeRaw()` unchanged. Apply bold regex BEFORE italic regex (to handle `**` before `*`).

3. **Integration point:** In `writeFooterCaption()` (line ~993-1018), caption text segments are passed through `latexEscape()`. Replace with the new caption-aware function.

4. **Czech typography:** Still apply `czechTypography()` after escaping (non-breaking spaces after single-letter prepositions).

### Suggested implementation

```go
// latexEscapeCaption escapes LaTeX specials but preserves ~ as non-breaking space
// and supports **bold** and *italic* inline formatting.
func latexEscapeCaption(s string) string {
    // Step 1: Escape LaTeX specials EXCEPT ~
    // (copy latexEscapeRaw replacer without the ~ line)
    escaped := latexEscapeCaptionRaw(s)
    
    // Step 2: Apply bold/italic (bold first to handle ** before *)
    escaped = regexp.MustCompile(`\*\*(.+?)\*\*`).ReplaceAllString(escaped, `\textbf{$1}`)
    escaped = regexp.MustCompile(`\*(.+?)\*`).ReplaceAllString(escaped, `\textit{$1}`)
    
    // Step 3: Czech typography
    return czechTypography(escaped)
}
```

Pre-compile the regexes as package-level vars (like existing patterns in markdown.go).
