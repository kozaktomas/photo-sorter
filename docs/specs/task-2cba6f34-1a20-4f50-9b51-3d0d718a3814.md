# Detect Duplicate/Similar Texts in Book

Flag texts in the book that are too similar to each other, helping the user avoid repetitive descriptions.

## Requirements

### 1. Detection logic

Compare all texts in the book pairwise using normalized Levenshtein distance (no AI needed, pure frontend computation):
- Normalize texts: lowercase, trim whitespace, remove punctuation
- Calculate similarity ratio (0.0 to 1.0) between all pairs
- Flag pairs with similarity > 0.6 (60%) as potential duplicates

For books with many texts, this is O(n²) but n is small (typically <100 texts), so it's fast enough in the browser.

### 2. Display on Texts tab

Add a "Duplikáty" (Duplicates) section or expandable panel on the Texts tab, below the text list:
- Show count in stats bar: "Možné duplikáty: 3" as a fourth stat card
- List each pair of similar texts:
  - Text A preview (truncated ~80 chars) + source (section/page)
  - Text B preview (truncated ~80 chars) + source (section/page)
  - Similarity percentage (e.g. "78% podobnost")
  - Links to navigate to each text ("Přejít")
- If no duplicates found, show "Žádné duplikáty" message

### 3. Implementation

Pure frontend — no backend changes needed. Compute similarity from the already-loaded book data (section photo descriptions + page slot text_content).

Use a simple Levenshtein distance implementation (or install a small library). Alternatively, use trigram-based similarity which is faster for longer texts.

### 4. i18n

Add keys:
- `books.editor.duplicateTexts`: "Možné duplikáty" (cs) / "Possible duplicates" (en)
- `books.editor.noDuplicates`: "Žádné duplikáty" (cs) / "No duplicates" (en)
- `books.editor.similarity`: "{{percent}}% podobnost" (cs) / "{{percent}}% similarity" (en)
