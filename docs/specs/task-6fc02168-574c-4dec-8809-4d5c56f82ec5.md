## Problem

In the Book Editor, when browsing pages and sections, photos are displayed as thumbnails without any identifying text. It's hard to know which photo is which without seeing the original filename or PhotoPrism title. This applies to:

- **Pages tab**: Page slots showing photo thumbnails (PageSlot.tsx, PageTemplate.tsx)
- **Sections tab**: Section photo pool (SectionPhotoPool.tsx)
- **Unassigned pool**: UnassignedPool.tsx

## Solution

Show the original filename (or PhotoPrism title if available) as an overlay or tooltip on photo thumbnails throughout the Book Editor.

### What to display

Use the photo's `FileName` or `Title` field from the API response. Priority:
1. `Title` if non-empty and not auto-generated (PhotoPrism sometimes sets title = filename stem)
2. `FileName` (original filename) as fallback

Display as a small text overlay at the bottom of the thumbnail (truncated with ellipsis if too long) and/or as a tooltip on hover.

### Where to display

1. **PageSlot.tsx** — Show filename below or overlaid on the photo thumbnail in page layout view
2. **PageTemplate.tsx** — Mini preview might be too small for text, but add a tooltip on hover
3. **SectionPhotoPool.tsx** — Show filename on each photo in the section's photo pool
4. **UnassignedPool.tsx** — Show filename on unassigned photos
5. **PhotoBrowserModal.tsx** — If not already showing filename, add it
6. **PhotoActionOverlay.tsx / PhotoInfoOverlay.tsx** — These overlays already exist on photos in the editor; add filename info here if not already present

### Data availability

Check whether the photo objects returned by these endpoints already include `FileName`/`Title`:
- `GET /api/v1/sections/{id}/photos` — returns section photos (check `SectionPhoto` type)
- `GET /api/v1/books/{id}` — returns book with pages/slots (check `PageSlot` type with photo data)
- `GET /api/v1/photos/{uid}` — full photo object has `FileName` and `Title`

If the section photos or page slot responses don't include filename, the backend handlers or types need to be extended. Check:
- `internal/web/handlers/books.go` — how slots are populated
- `internal/database/types.go` — `PageSlot` struct
- `web/src/types/index.ts` — frontend types

The photo data might need to be enriched with a PhotoPrism lookup if only `photo_uid` is stored in the slot.

### UI Design

- Small, semi-transparent dark overlay at bottom of thumbnail with white text (similar to how PhotoInfoOverlay already works)
- Text truncated with CSS `text-overflow: ellipsis` and `overflow: hidden`
- Tooltip (`title` attribute) with full filename on hover
- Font size ~10-11px so it doesn't overwhelm the thumbnail
- Don't show on text-only slots
