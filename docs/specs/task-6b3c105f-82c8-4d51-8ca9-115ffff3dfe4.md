## Goal

In the Book Editor → Pages tab, add the ability to export just the currently selected/edited page as a PDF. This allows the user to quickly preview how a single page will look in the final print output without exporting the entire book.

## UX

- Add a button (e.g. PDF icon or "Export page" button) in the page sidebar or page editor area for the currently selected page
- Clicking it opens the PDF directly in the browser (inline view), NOT as a download
- Should be fast since it's only one page (no need for the full book export flow)

## Backend

### New API endpoint
`GET /api/v1/pages/{id}/export-pdf` — export a single page as PDF.

**Response headers:**
- `Content-Type: application/pdf`
- `Content-Disposition: inline` (NOT `attachment`) — this makes the browser open the PDF in its built-in viewer instead of downloading it

Implementation:
1. Load the page with its slots (photos + text) from the database
2. Load the section and chapter (for chapter color on H1 headings)
3. Load the book (for book-level settings like title)
4. Reuse the existing LaTeX generation from `internal/latex/latex.go` but generate only the one page instead of the whole book
5. Run lualatex and return the PDF

The existing `internal/latex/` package generates a full book. Options:
- **Option A (preferred):** Add a function like `GenerateSinglePagePDF(page, book)` that produces a minimal LaTeX document with just the one page — same layout, fonts, dimensions, but only one page content
- **Option B:** Generate the full book LaTeX but with only the selected page's data passed in

Option A is better for performance (much less data to process, faster lualatex run).

The single-page PDF must use the exact same layout, fonts, margins, and styling as the full book export — otherwise it defeats the purpose of previewing.

### Frontend
- `web/src/pages/BookEditor/PagesTab.tsx` or `PageSidebar.tsx` — add export button for selected page
- On click: open the endpoint URL in a new browser tab (`window.open(url, '_blank')`) — the browser will display the PDF inline thanks to the `Content-Disposition: inline` header
- Show loading indicator while generating (optional — since it opens in a new tab, the tab itself will show loading)

## Important

- Reuse existing LaTeX templates and styling — the single page must look identical to how it appears in the full book
- Include proper even/odd page margin handling (inside/outside margins depend on page position). Either use the page's actual position in the book, or default to a reasonable side.
- Work-in-progress book exists — no schema changes, purely new endpoint + UI button
