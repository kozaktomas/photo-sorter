# MCP Text Tools for Photo Book

Add text-related MCP tools to the existing MCP server. These tools allow AI to check, rewrite, and manage text content in the photo book, as well as check style consistency across all texts.

## Prerequisites

The MCP server infrastructure exists in `internal/mcp/`. Book, chapter, section, page, and slot tools are already implemented.

## MCP Tools — AI Text Operations

1. **`check_text`** — AI-powered text check (spelling, grammar, diacritics)
   - Parameters: `text` (string, required), `source_type` (string, optional — `slot`/`photo_description`/`photo_note`), `source_id` (string, optional), `field` (string, optional)
   - Returns: check result with suggestions, readability score, and corrected text
   - When `source_type`, `source_id`, and `field` are provided, the result is also persisted to the database (same as the `check-and-save` endpoint)

2. **`rewrite_text`** — AI-powered text rewrite (length adjustment)
   - Parameters: `text` (string, required), `target_length` (string, required — one of: `much_shorter`, `shorter`, `longer`, `much_longer`)
   - Returns: rewritten text

3. **`check_consistency`** — AI-powered style consistency check across all book texts
   - Parameters: `book_id` (int, required)
   - Returns: consistency analysis with suggestions for each text

## MCP Tools — Text Version History

4. **`list_text_versions`** — List version history for a text field
   - Parameters: `source_type` (string, required), `source_id` (string, required), `field` (string, required)
   - Returns: array of versions with id, content, changed_by (user/ai), created_at

5. **`restore_text_version`** — Restore a previous text version
   - Parameters: `version_id` (int, required)
   - Returns: restored text content

## Implementation Notes

- Add tools in `internal/mcp/text.go`
- Register tools in the existing server setup
- For AI text operations, reuse the existing AI provider (OpenAI) initialization from config
- The `check_text` tool should use the existing `TextCheckAndSave` handler logic for persistence
- The `rewrite_text` tool wraps the existing AI rewrite functionality
- The `check_consistency` tool needs to gather all texts from the book (photo descriptions + text slots) and send them to the AI — reuse the existing logic from the `TextHandler`
- Include cost estimates in the response if available (the existing text handlers return token usage)
